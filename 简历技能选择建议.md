# 简历中性能测试工具选择建议：JMeter vs k6

## 快速答案

**最佳策略**：两者都写，但根据目标公司类型调整优先级

---

## 市场认知度对比

### JMeter 的优势

✅ **知名度高**：几乎所有 HR 和面试官都认识  
✅ **传统企业认可**：银行、保险、传统软件公司更熟悉  
✅ **简历通过率高**：关键词匹配度高，容易被筛选  
✅ **岗位需求多**：招聘网站上 JMeter 岗位更多  

### k6 的优势

✅ **技术前沿**：体现你关注新技术趋势  
✅ **互联网公司青睐**：字节、阿里、腾讯等大厂更认可  
✅ **差异化优势**：会 k6 的人相对较少，竞争小  
✅ **技术深度**：体现对现代工具链的理解

## 不同公司类型的建议

### 1. 传统企业（银行、保险、制造业）

**推荐写法**：
```
性能测试工具：JMeter、k6
```

**理由**：
- JMeter 是必须项，传统企业更认可
- k6 作为加分项，体现学习能力
- 两者结合展示技术广度

### 2. 互联网公司（字节、阿里、腾讯、美团等）

**推荐写法**：
```
性能测试工具：k6、JMeter
```

**理由**：
- k6 更符合云原生、CI/CD 趋势
- 体现对现代工具链的理解
- JMeter 作为备选技能

### 3. 创业公司 / 中小型互联网公司

**推荐写法**：
```
性能测试工具：k6（主要）、JMeter
```

**理由**：
- k6 轻量级，适合快速迭代
- 资源占用低，成本友好
- 完美支持 CI/CD，符合敏捷开发

### 4. 外企 / 跨国公司

**推荐写法**：
```
性能测试工具：JMeter、k6、Gatling
```

**理由**：
- 展示工具多样性
- 体现对不同场景的理解
- 外企更看重技术广度

**优点**：
- 展示实际应用能力

## 面试准备建议

### 如果写 JMeter，需要准备：

1. **基础问题**：
   - JMeter 的工作原理
   - 如何设置线程组和采样器
   - 如何分析测试结果

2. **进阶问题**：
   - 分布式测试如何配置
   - 如何自定义插件
   - 性能优化技巧

3. **实战问题**：
   - 遇到过哪些性能问题
   - 如何定位性能瓶颈
   - 测试报告如何解读

### 如果写 k6，需要准备：

1. **基础问题**：
   - k6 的优势是什么
   - 如何编写测试脚本
   - 如何设置阈值

2. **进阶问题**：
   - 如何集成到 CI/CD
   - 如何输出到 Grafana
   - 如何优化脚本性能

3. **实战问题**：
   - 实际项目中的应用场景
   - 与 JMeter 的对比
   - 遇到的挑战和解决方案

## 常见问题 FAQ

### Q1: 如果只会 k6，不会 JMeter 怎么办？

**建议**：
- 快速学习 JMeter 基础（1-2周即可）
- 简历上写"k6（精通）、JMeter（了解）"
- 面试时诚实说明，强调学习能力

### Q2: 如果只会 JMeter，不会 k6 怎么办？

**建议**：
- 如果目标互联网公司，建议学习 k6（1周可上手）
- 如果目标传统企业，JMeter 足够
- 简历上可以写"JMeter（熟练），正在学习 k6"

### Q3: 两个工具都写会不会显得不够专业？

**不会**。相反，这体现：
- ✅ 技术广度
- ✅ 适应不同场景的能力
- ✅ 持续学习的态度

### Q4: 哪个工具更容易找到工作？

**JMeter**：
- 岗位数量更多
- 但竞争也更激烈
- 薪资相对较低

**k6**：
- 岗位数量较少
- 但竞争小
- 薪资相对较高
- 增长趋势明显

### Q5: 如何快速学习另一个工具？

**从 JMeter 到 k6**：
- 时间：1-2 周
- 重点：JavaScript 基础、k6 API
- 资源：官方文档 + 示例脚本

**从 k6 到 JMeter**：
- 时间：2-3 周
- 重点：GUI 操作、测试计划结构
- 资源：官方文档 + 视频教程
